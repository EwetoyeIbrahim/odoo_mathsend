Point of Sale Default Opening Cash to Zero

Always set default opening cash to zero and allow but still allow manual input.

About

The problem and solution

The Problem:
The point of sale module assumes a cashier will always open the day with the previous closing cash and as such sets the opening cash as the previous session closing cash. Although this can always be changed manually, Cashiers typically forget to change the value which eventually leads to accounting issues, especially with the profit/loss statement.

The Solution:
In some business cases, it will be more rational to set the default to zero and have the cashiers change the figures when necessary.

Features

What this module does

Defaults the point of sale opening cash to 0
